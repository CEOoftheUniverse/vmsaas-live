🔐 Infrastructure Security

  • Each customer gets a dedicated Windows VM — no shared containers or multi-tenant compute
  • All data encrypted at rest using AES-256
  • All connections secured with TLS 1.3
  • VM access via RDP with unique credentials — only you have the password
  • Network-level isolation between customer VMs
  • Regular security patches and OS updates applied automatically

🛡️ Data Privacy

  • Your code, prompts, and agent outputs are never used to train AI models
  • LLM API calls are made from your VM, not through our infrastructure
  • ChromaDB memory stores are local to your VM only
  • No telemetry or analytics collected from agent activity
  • GDPR and CCPA compliant — full data deletion on request

🔑 Access Control

  • Full admin access to your VM — you control everything
  • MoltBot staff cannot access your VM without explicit permission
  • API keys stored with envelope encryption
  • Webhook endpoints require HMAC signature verification
  • Rate limiting on all public API endpoints

📊 Monitoring & Compliance

  • Health checks run every 60 seconds with auto-restart on failure
  • Public status page at status.moltbot.cloud
  • 99.5% uptime SLA on all plans
  • Incident response within 4 hours (Enterprise: 1 hour)
  • SOC 2 Type II certification in progress (target Q3 2026)

🤝 Our Security Promise

We believe your code is your most valuable asset. That's why every MoltBot VM is fully isolated, every byte is encrypted, and we will never use your data to train models. If we ever change this policy, you'll get 90 days notice and a full refund.
