Vector Databases Explained: Pinecone, Weaviate, pgvector & How to Choose

Every RAG system needs a vector store. The choice between a managed vector database and pgvector in your existing Postgres has major implications for cost, scale, and operational complexity. Here's how to decide.

A vector database stores high-dimensional embeddings — numerical representations of text, images, or other data — and enables fast approximate nearest-neighbor (ANN) search to find the most semantically similar items given a query embedding. They're the engine behind every RAG pipeline.

How vector search works

When you embed a query ("what's our refund policy?"), you get a vector like [0.23, -0.71, 0.44, ...]. The vector DB finds the stored chunks with the smallest cosine distance from that query vector. The closest chunks are your most semantically relevant results — regardless of keyword overlap.

When to use pgvector

  • Your corpus is under 5–10M vectors and doesn't need sub-10ms latency at scale.
  • You already run Postgres and want to avoid another managed service.
  • You need rich SQL filtering (e.g., WHERE tenant_id = 'acme' AND created_at > '2026-01-01') alongside vector search.
  • You're building an MVP — pgvector lets you start fast and migrate later if needed.

Connecting to your vector store on MoltBot

from moltbot.rag import RAGPipeline from moltbot.vector_stores import Pinecone, Weaviate, PGVector # Choose your vector store — same API regardless vector_store = Pinecone( index_name="production-docs", api_key=os.environ["PINECONE_API_KEY"] ) pipeline = RAGPipeline( vector_store=vector_store, embedding_model="text-embedding-3-large", top_k=5, reranker="cohere-rerank-3" # optional reranking step ) result = pipeline.query("What is our enterprise SLA?")

Decision guide

  • Start with pgvector if you're early-stage — it's free, uses existing infrastructure, and works well at modest scale.
  • Upgrade to Pinecone or Qdrant when you exceed 10M vectors, need sub-20ms p99 latency, or are building a multi-tenant SaaS.
  • Use Weaviate for hybrid keyword+semantic search or multi-modal (text + image) retrieval.
