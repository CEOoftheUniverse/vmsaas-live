AI Agent Memory Systems: ChromaDB, Embeddings & Persistent Context

Stateless agents forget everything between sessions. Stateful agents learn, adapt, and compound knowledge over time. Here's how to build the memory layer that makes the difference — with production code.

The biggest gap between demo agents and production agents isn't model quality — it's memory. Demo agents start fresh every session. Production agents remember every past decision, every codebase quirk, every user preference. The infrastructure to make that happen is surprisingly straightforward.

The 4 Types of Agent Memory

Before building, understand what you're building. Agent memory systems typically handle four categories:

  • Working memory — the current conversation context (in-window). Limited by model context length.
  • Episodic memory — past actions, decisions, and outcomes. "What did I do last time?"
  • Semantic memory — facts about the world, the user, and the codebase. "What does this function do?"
  • Procedural memory — learned workflows and preferences. "The user prefers pytest over unittest."

ℹ️ Architecture insight

MoltBot agents use ChromaDB for episodic + semantic memory, Redis for fast working-memory overflow, and a SQLite preference store for procedural memory. Three different systems, unified behind a single Memory API.

ChromaDB: The Core Memory Store

ChromaDB is an open-source vector database — you store embeddings of text (memories), then retrieve the most semantically similar memories at query time. Perfect for episodic and semantic memory.

import chromadb
from chromadb.config import Settings

# Initialize persistent ChromaDB
client = chromadb.PersistentClient(
    path="/data/agent_memory",
    settings=Settings(anonymized_telemetry=False)
)

# One collection per agent (or per user)
memory = client.get_or_create_collection(
    name="agent_001",
    metadata={"hnsw:space": "cosine"}
)

Storing Memories

Every significant agent action, observation, or user instruction gets stored as an embedding with metadata.

from datetime import datetime
import hashlib

def store_memory(agent_id: str, text: str, memory_type: str, metadata: dict = {}):
    """Store a memory with embedding and metadata."""
    collection = client.get_collection(f"agent_{agent_id}")
    
    # Generate stable ID from content hash
    doc_id = hashlib.sha256(text.encode()).hexdigest()[:16]
    
    collection.add(
        documents=[text],
        metadatas=[{
            "type": memory_type,   # "episodic" | "semantic" | "procedural"
            "timestamp": datetime.utcnow().isoformat(),
            "importance": metadata.get("importance", 0.5),
            **metadata
        }],
        ids=[doc_id]
    )

# Example usage
store_memory("001", 
    "User prefers concise code with type hints. Rejected verbose example.",
    "procedural",
    {"importance": 0.9, "topic": "code_style"}
)

Retrieving Relevant Memories

At the start of each task, query ChromaDB for the most relevant past memories and inject them into the prompt as context.

def recall(agent_id: str, query: str, n: int = 5, memory_type: str = None) -> list:
    """Retrieve N most relevant memories for a given query."""
    collection = client.get_collection(f"agent_{agent_id}")
    
    where_filter = {"type": memory_type} if memory_type else None
    
    results = collection.query(
        query_texts=[query],
        n_results=n,
        where=where_filter
    )
    
    return [
        {"text": doc, "metadata": meta, "distance": dist}
        for doc, meta, dist in zip(
            results["documents"][0],
            results["metadatas"][0],
            results["distances"][0]
        )
    ]

# Inject memories into system prompt
memories = recall("001", "writing unit tests for auth module")
memory_block = "\n".join([f"- {m['text']}" for m in memories])
system_prompt = f"Relevant context from past sessions:\n{memory_block}\n\nNow: {task}"

Hierarchical Memory Compression

Raw memories accumulate fast. Without compression, retrieval degrades. Use hierarchical summarization to keep the memory store lean.

def compress_old_memories(agent_id: str, days_old: int = 7):
    """Summarize memories older than N days into compressed episodes."""
    cutoff = (datetime.utcnow() - timedelta(days=days_old)).isoformat()
    
    collection = client.get_collection(f"agent_{agent_id}")
    old_memories = collection.get(
        where={"timestamp": {"$lt": cutoff}}
    )
    
    if len(old_memories["documents"]) < 10:
        return  # Not enough to compress
    
    # Summarize with a cheap, fast model
    summary = call_model("gemini-flash",
        f"Summarize these agent memories into key learnings:\n"
        + "\n".join(old_memories["documents"])
    )
    
    # Store summary, delete originals
    store_memory(agent_id, summary, "episodic_compressed", 
                 {"importance": 0.8, "original_count": len(old_memories["documents"])})
    
    collection.delete(ids=old_memories["ids"])

Importance Scoring

Not all memories are equal. Assign importance scores so lower-ranked memories are compressed or evicted first.

def score_importance(event: str) -> float:
    """Score memory importance 0-1 using a fast classifier."""
    keywords = {
        "user preference": 0.9, "bug fix": 0.8, "architecture decision": 0.85,
        "error": 0.7, "success": 0.6, "routine": 0.3
    }
    score = 0.5  # Default
    for keyword, weight in keywords.items():
        if keyword in event.lower():
            score = max(score, weight)
    return score

Memory-Augmented Prompting

Full pattern: retrieve relevant memories, format them, inject into the system prompt, then run the agent. This is the full loop MoltBot uses.

async def run_agent_with_memory(agent_id: str, task: str, model: str):
    # 1. Recall relevant memories
    episodic = recall(agent_id, task, n=3, memory_type="episodic")
    procedural = recall(agent_id, task, n=3, memory_type="procedural")
    
    # 2. Build memory-augmented prompt
    system = f"""You are an autonomous coding agent.

Past relevant experiences:
{format_memories(episodic)}

User preferences and learned behaviors:
{format_memories(procedural)}

Use these memories to inform your approach, but don't repeat past mistakes."""

    # 3. Run agent
    result = await call_model_async(model, task, system_prompt=system)
    
    # 4. Store new memories from this session
    store_memory(agent_id, f"Task: {task}. Outcome: {result.summary}", 
                 "episodic", {"importance": score_importance(result.summary)})
    
    return result
